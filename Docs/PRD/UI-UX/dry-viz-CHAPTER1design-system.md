## 1. Purpose

The Design System defines the visual language of Dry Viz.

Every page, component, animation, spacing rule, typography rule, and color must follow this document.

This ensures consistency throughout the application.

---

## 2. Design Philosophy

Dry Viz should feel

- Modern
- Premium
- Professional
- Educational
- Interactive
- Minimal
- Fast

Users should immediately feel that they are using a professional developer tool.

## 7. Shadows

Use soft shadows only.

Avoid heavy shadows.

The interface should feel clean.

---

## 8. Icons

Use a single icon library across the application.

Icons should be simple, outlined, and consistent.

---

## 9. Animations

Animations should feel smooth.

Every animation must have a purpose.

Never animate for decoration alone.

Examples

- Panel opening
- Variable update
- Memory allocation
- Timeline movement
- Graph transitions

## 17. Design Principles

Clarity over decoration.

Consistency over creativity.

Performance over complexity.

Every element must help users learn.

---

## 18. Final Goal

Dry Viz should feel like a premium developer platform rather than a traditional educational website.

The interface should inspire confidence while keeping the learning experience simple and distraction-free.
