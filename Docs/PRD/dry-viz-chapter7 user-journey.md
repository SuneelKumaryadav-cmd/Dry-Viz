## 14. User Goals

Every user journey should help the user achieve one or more of the following goals.

- Understand code execution
- Learn programming visually
- Improve algorithm knowledge
- Understand Time Complexity
- Understand Space Complexity
- Learn from AI
- Save learning progress

---

## 15. Exit Points

Users may leave Dry Viz after:

- Completing visualization
- Saving project
- Sharing visualization
- Finishing learning session
- Logging out

The application should preserve progress whenever possible.

---

## 16. Future User Journeys

Future versions may include:

- Classroom Learning Journey
- Team Collaboration Journey
- AI Mentor Journey
- University Dashboard Journey
- Coding Challenge Journey
- Interview Preparation Journey

---

## 17. Journey Principles

Every user journey should be:

Simple
Fast
Visual
Interactive
Educational
Predictable

Every click should move the learner closer to understanding the code.

---

## 18. Success Criteria

A successful user journey means:

- Users can start within minutes.
- Users understand code without confusion.
- Users complete visualizations successfully.
- Users can replay and review concepts.
- Users leave the platform with a better understanding than when they arrived.
